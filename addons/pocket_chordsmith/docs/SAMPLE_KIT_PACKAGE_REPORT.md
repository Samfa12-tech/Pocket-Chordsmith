## Generation Source

The kit is generated by `editor/pcs_sound_kit_generator.gd` through the editor `Generate Web Sound Kit` action. The generator writes the WAV files and the HYBRID profile into:

```text
res://addons/pocket_chordsmith/audio/web_kit/
```

The generated profile maps Pocket Chordsmith drum, lofi, chip, bass, chord, melody, guitar, accent, and stinger IDs to these preview assets. Regenerate the kit after changing shared sound constants or generator mappings, then rerun the package-size measurement before a release.

## Import Metadata

The addon packager excludes `.uid` files and excludes `.import` files unless `--include-import-metadata` is explicitly passed. This keeps shipped addon ZIPs portable and lets Godot rebuild project-local import caches after install.

Asset Library submissions should use the default package behavior. Do not include generated `.godot/`, `.import`, `.uid`, export builds, or local cache files in the addon payload.

## License And Shipping Guidance

The generated preview kit is part of the Pocket Chordsmith Godot addon and follows the addon license in `addons/pocket_chordsmith/LICENSE` unless a future release says otherwise.

For shipped games:

- Use `STEM_SYNC` with rendered stems for production music beds when possible.
- Use `HYBRID` for stems plus event-triggered accents, stingers, or sample hits.
- Replace preview WAVs with your own licensed production samples when audio identity or platform constraints matter.
- Keep preview kit usage clear in release notes if a game ships these bundled sounds unchanged.
- Recheck package size after adding stems, loops, or higher-fidelity samples; the current preview kit is intentionally small enough for addon onboarding.
