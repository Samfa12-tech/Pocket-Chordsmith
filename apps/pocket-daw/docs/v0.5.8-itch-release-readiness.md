# Pocket DAW v0.5.8 Itch Release Readiness

Pocket DAW v0.5.8 is the free Windows desktop updater-test pass focused on timeline sequencer alignment and direct song settings. The persisted `.pocketdaw` schema remains version 2.

## Release Target

- Primary itch artifact: `releases/itch/pocket-daw-windows-x64-v0.5.8/`
- Manual ZIP archive: `releases/itch/pocket-daw-windows-x64-v0.5.8.zip`
- Optional installers: `releases/itch/installers/`
- Butler channel: `windows-x64`
- Browser preview: development/preview only, not the main itch target.

## What Changed

- Timeline inline sequencer boxes now start exactly at the section/bar edge with no label-column offset.
- Drum, bass, melody and guitar lane labels now live in the left track header so the editable grid starts on bar 1.
- Inline sequencer boxes are generated from the active song time signature, section length and sequencer resolution.
- The timeline toolbar now exposes direct BPM, key, scale, time signature, sequencer resolution and Add Section controls.
- The default timeline zoom now opens much closer at 240 px/bar, while the live zoom slider and ruler-over-row layering remain in place.
- Release artifact generation still writes checksum and manifest files from the actual staged files.

## Signing And Smoke Status

- Authenticode signing is not claimed unless `Get-AuthenticodeSignature` verifies it.
- Unsigned builds may trigger Windows SmartScreen warnings.
- Manual Windows smoke status is `NOT RUN` until `WINDOWS_SMOKE_CHECKLIST_v0.5.8.md` is completed against the exact artifact hash.

## Commands

```powershell
npm ci
npm run verify:itch
```

Preview upload command, do not run without manual approval:

```powershell
butler push-preview releases/itch/pocket-daw-windows-x64-v0.5.8 samfa12/pocket-daw:windows-x64
```

Hidden upload command, do not run without manual approval:

```powershell
butler push releases/itch/pocket-daw-windows-x64-v0.5.8 samfa12/pocket-daw:windows-x64 --userversion 0.5.8 --hidden
```

If the itch slug differs, replace `samfa12/pocket-daw`.

## Known Limitations

- No live recording yet.
- No ASIO backend yet.
- Imported audio decode/streaming is still limited; oversized files are rejected with friendly errors.
- Godot/web exports are manifest previews unless full asset-pack assembly is implemented.
- Stem export is sequential rather than a bundled stem ZIP.
- Native cache hydration skips stale, invalid or partial generated-stem cache groups.
- Automation is limited to first-pass track volume and pan.
- Send/return processing is scaffolded and guarded.
- No claim of professional DAW completeness.
