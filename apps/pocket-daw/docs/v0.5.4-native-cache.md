# Pocket DAW v0.5.4 Native Cache Notes (Historical)

v0.5.4 was a native-cache foundation pass for the installed Pocket DAW path. This document is historical and must not be used as the current release checksum source. Current itch release hashes are generated under `releases/itch/` by `npm run package:itch`.

## Implemented

- Generated Chordsmith section clips can render drums, bass, chords, melody and guitar stem WAV assets for native playback regions.
- Runtime-loaded audio clips can be encoded into WAV assets and added to the native region payload when their decoded buffers are available.
- Cache assets have stable project-relative paths under `project-cache/native-audio/`.
- The installed app exposes guarded Tauri commands to write/read native cache WAV assets beside a saved `.pocketdaw` project.
- Build Native Cache writes WAV assets, merges optional `project.renderCache` metadata, and saves the project refs.
- Diagnostics report cached asset/region counts, cache metadata count, hit/miss counts, procedural fallback events and runtime-audio cache misses.

## Still Next At v0.5.4

- Move imported audio decode/streaming fully into Rust/native code.
- Add worker/Rust-side cache generation so large projects do less CPU-heavy work on the UI thread.
- Add cache cleanup for stale source hashes once real-project QA confirms the file layout.

## v0.5.5 Follow-Up

v0.5.5 adds guarded persisted native cache hydration for valid `project-cache/native-audio/*.wav` entries when source hashes match. Stale, invalid-path or partial generated-stem cache groups are skipped and reported in diagnostics.
