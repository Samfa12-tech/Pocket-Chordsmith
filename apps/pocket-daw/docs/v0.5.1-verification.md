## Manual Windows Installer Checklist

| Area | Status | Steps |
| --- | --- | --- |
| Installer starts | NOT RUN | Launch `Pocket DAW_0.5.1_x64-setup.exe` from the uploaded package. Confirm Windows allows installation or records the SmartScreen/signing warning. |
| App launches after install | NOT RUN | Open Pocket DAW from Start Menu or installer finish screen. Confirm the first DAW shell renders and no blank window appears. |
| Version display | NOT RUN | Confirm the transport shows `v0.5.1` and the environment label reports native/Tauri. |
| Native open | NOT RUN | File > Open a `.pocketdaw` project. Confirm project title, timeline, tracks, media pool and recent project label update. |
| Native save | NOT RUN | Save an edited project, close, reopen, and verify the edit persisted. |
| Save As | NOT RUN | Save to a new `.pocketdaw` path and confirm the current file label changes. |
| Audio import dialog | NOT RUN | Import `.wav` or another supported audio file. Confirm media status is runtime-loaded, external unloaded after reload, and no silent failure. |
| MIDI import dialog | NOT RUN | Import `.mid`/`.midi`; confirm a MIDI clip appears and piano-roll basics work. |
| Playback | NOT RUN | Play, pause, stop, scrub, loop selected clip, and confirm meters move without UI overlap. |
| Export full WAV/MIDI | NOT RUN | Export full WAV and MIDI; confirm files are created and non-empty. |
| Stem export | NOT RUN | Export stems; confirm sequential downloads/files are named deterministically. |
| Manifest exports | NOT RUN | Export section/Godot/web manifests; confirm warnings are honest and paths match `audio/full`, `audio/stems`, `audio/sections`, `manifests`. |
| Diagnostics | NOT RUN | Export diagnostics and confirm version, environment, file label, playback and audio scheduler data are present. |

## Known Limitations

- Native relink/reload and collect-media copy remain guarded. The app can export a collect-media plan, but it does not yet copy files into a project media folder.
- Browser imports are runtime-only unless the native shell provides a durable path.
- Section-loop and game-pack manifests use deterministic pack paths, but section-loop audio rendering and ZIP assembly are still planned-render follow-up work.
- Recording controls are visible only as disabled/stubbed language. No microphone or instrument capture is implemented.
- Pocket Audio Core comparison is adapter-only because the real core package/API is not present in this checkout.

## Private Alpha Blockers
