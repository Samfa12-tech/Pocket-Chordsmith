# Pocket DAW v0.5.5 Itch Release Readiness

Pocket DAW v0.5.5 is the free Windows desktop itch-release-readiness pass. The persisted `.pocketdaw` schema remains version 2.

## Release Target

- Primary itch artifact: `releases/itch/pocket-daw-windows-x64-v0.5.5/`
- Manual ZIP archive: `releases/itch/pocket-daw-windows-x64-v0.5.5.zip`
- Optional installers: `releases/itch/installers/`
- Butler channel: `windows-x64`
- Browser preview: development/preview only, not the main itch target.

## What Changed

- Added portable Windows package automation with release docs, checksum manifest and JSON release manifest.
- Added artifact verification so documented SHA-256 hashes are recomputed from actual files.
- Added untrusted project rendering hardening for string-rendered HTML, data attributes and inline colors.
- Added loaded-project normalization for malformed tracks, clips, media, markers and automation lanes.
- Added native and browser fallback file-size limits before whole-file reads.
- Added persisted native cache hydration for valid `project-cache/native-audio/*.wav` entries when source hashes match.

## Signing And Smoke Status

- Authenticode signing is not claimed unless `Get-AuthenticodeSignature` verifies it.
- Unsigned builds may trigger Windows SmartScreen warnings.
- Manual Windows smoke status is `NOT RUN` until `WINDOWS_SMOKE_CHECKLIST_v0.5.5.md` is completed against the exact artifact hash.

## Commands

```powershell
npm ci
npm run verify:itch
```

Preview upload command, do not run without manual approval:

```powershell
butler push-preview releases/itch/pocket-daw-windows-x64-v0.5.5 samfa12/pocket-daw:windows-x64
```

Hidden upload command, do not run without manual approval:

```powershell
butler push releases/itch/pocket-daw-windows-x64-v0.5.5 samfa12/pocket-daw:windows-x64 --userversion 0.5.5 --hidden
```

If the itch slug differs, replace `samfa12/pocket-daw`.

## Known Limitations

- No live recording yet.
- No ASIO backend yet.
- Imported audio decode/streaming is still limited; oversized files are rejected with friendly errors.
- Godot/web exports are manifest previews unless full asset-pack assembly is implemented.
- Stem export is sequential rather than a bundled stem ZIP.
- Native cache hydration skips stale, invalid or partial generated-stem cache groups.
- Automation is limited to first-pass track volume and pan.
- Send/return processing is scaffolded and guarded.
- No claim of professional DAW completeness.
