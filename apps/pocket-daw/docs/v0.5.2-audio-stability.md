# Pocket DAW v0.5.2 Audio Stability Notes

Date: 2026-06-12

v0.5.2 is a desktop-audio pivot. Pocket DAW is an installable computer app, not a Web Audio product, so generated Pocket Chordsmith playback now has a native CPAL path in the Tauri layer. Web Audio remains only a browser/dev fallback and for unfinished runtime-only paths that have not moved native yet.

## What Changed

- Added Tauri native audio commands backed by CPAL:
  - `native_audio_start`
  - `native_audio_pause`
  - `native_audio_seek`
  - `native_audio_stop`
  - `native_audio_status`
  - `native_audio_update_track`
- Added a TypeScript native playback bridge in `src/native/audioPlayback.ts`.
- `AudioEngine.play()` now tries native CPAL playback first for rendered generated/MIDI events.
- Generated Pocket Chordsmith drums, bass, chords, melody, guitar and MIDI-preview events are sent to the native event synth in the installed app.
- Web Audio lookahead playback is still available as a fallback when the Tauri runtime or native command path is unavailable.
- Mixer volume, pan, mute and solo changes are forwarded to the native engine without calling `setProject()`.
- `applyProjectState()` now supports explicit audio sync modes and avoids full project-load sync while playback is running on legacy edit paths.
- Late Web Audio fallback events now skip badly late drums/hats instead of bunching them at `currentTime`.

## Diagnostics Added

The diagnostics JSON now includes:

- `playbackBackend`
- `nativeAudio.status`
- `nativeAudio.lastError`
- `schedulerTickCount`
- `missedSchedulerTickCount`
- `maxSchedulerGapMs`
- `lateEventCount`
- `skippedLateEventCount`
- `audioGraphReconfigureCount`
- `activeAudioSourcesStoppedByGraphReconfigureCount`
- `projectSyncCount`
- `lastProjectSyncMode`
- `lastProjectSyncReason`
- `lastAudioDropCause`
- `ui.renderCountDuringPlayback`

## Audio-Safe Sync Paths

- Mixer control previews and commits stay on the mixer fast path and update existing native track controls.
- Chordsmith composition edits use an event/timeline resync path during playback instead of a hard project-load sync.
- Full project load is still allowed to rebuild playback state.
- Mixer graph topology and FX replacement are still not a full crossfaded native graph implementation.

## Remaining Limitations

- The native CPAL path is a first generated-event playback engine, not a parity instrument engine yet.
- Imported decoded audio clips still depend on the current Web Audio buffer cache for playback/export when those buffers are available.
- Audio-file import still reads bytes through native dialogs but decodes with `AudioContext.decodeAudioData`; WAV/MP3/FLAC/OGG/AIFF should not be called DAW-grade until native decoding lands.
- Native decoding should move to Rust through Symphonia or an equivalent decoder before audio-file support is advertised as release-grade.
- Native playback does not yet stream imported audio files from disk or persist decoded media across reload.
- Manual Windows installer QA has not been run for v0.5.2 in this pass.
