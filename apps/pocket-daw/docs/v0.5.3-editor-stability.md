# Pocket DAW v0.5.3 Editor Stability Notes

v0.5.3 is a stabilisation pass for the installed Tauri/native Pocket DAW path. The product direction remains desktop-first: native CPAL playback is preferred, while Web Audio stays available only as a browser/dev fallback.

## Fixed

- Chordsmith sequencer edits no longer reset the inspector to the top. The app now snapshots and restores the intentional scroll panes around full renders.
- The main app shell, track list, timeline scroll area, inspector, mixer and media pool have stable `data-scroll-key` anchors for scroll restoration.
- Chordsmith editor changes now use a `composition-events` sync path instead of the broad project-load/default path.
- Drum, bass and melody step-cell edits patch the current cell and selection state directly instead of replacing the whole shell on every click.
- Native CPAL playback bypasses stale WAV render-cache regions after live Chordsmith edits, then restarts from the current position with updated procedural events so new hits/notes are audible without waiting for a synchronous stem rebuild.
- Native render-cache prewarm can build clean generated-section stem assets while the installed app is idle, so the Play command uses a ready cache when available and falls back to procedural events when it is not.
- Runtime-loaded audio clips can be handed to native playback as WAV asset regions when decoded buffers are available.

## Diagnostics

The diagnostics export now includes native render-cache stability fields:

- `nativeRenderCache.assetRegionCount`
- `nativeRenderCache.renderCacheHitCount`
- `nativeRenderCache.renderCacheMissCount`
- `nativeRenderCache.proceduralFallbackEventCount`
- `nativeRenderCache.runtimeAudioRegionCount`
- `nativeRenderCache.missingRuntimeAudioRegionCount`
- `nativeRenderCache.prewarmScheduled`
- `nativeRenderCache.nativeRenderCacheBypassedForLiveEdits`
- `nativeRenderCache.buildCount`
- `nativeRenderCache.lastBuildMs`
- `nativeRenderCache.lastBuildReason`

## Known Limitation

The pre-rendered native cache can now prewarm outside the Play command, but it is still generated on the main app side and kept as runtime state. It should move to a worker, Rust path or persisted disk cache in the next performance pass so clean cache builds become durable and even less visible to the UI.

## Manual Smoke Focus

1. Launch the packaged Windows app.
2. Load the demo copy.
3. Select Drums, scroll the inspector to the Section Sequencer and click Kick/Snare/Hat steps.
4. Confirm the inspector stays at the same scroll position.
5. Press Play and repeat edits while playback runs.
6. Export diagnostics and confirm `playbackBackend` is `native-cpal` and `nativeRenderCache.nativeRenderCacheBypassedForLiveEdits` becomes true only after live composition edits.
