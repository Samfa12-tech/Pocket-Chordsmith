Run this against the exact installed Windows alpha from itch/GitHub release artifacts. Do not use an extracted portable app folder.

## Required Reliability Gate For Each New Public Candidate

Run and record these rows against the exact installed candidate before calling a build public-ready. `release-status.json` is the machine-readable release truth, and `docs/CURRENT_RELEASE_STATUS.md` is the generated human-readable release truth. Do not treat a source commit, historical checklist row, or local dev build as public release evidence unless those files record matching artifact and smoke evidence.

- Native Open: File -> Open accepts a valid `.pocketdaw` and a JSON project, updates the title/current path, populates recent files, and leaves a friendly error for invalid/corrupt files without closing the app.
- Native Save: Save writes to the current `.pocketdaw` path, survives app close/reopen, and preserves timeline clips, source refs, media metadata, automation, routing, and current project labels.
- Native Save As: Save As writes a new `.pocketdaw`, adopts the new path/title, updates recent files, and does not keep writing to the old path.
- Recent files: recently opened/saved native paths appear with useful labels, reopen from the menu, and stale/missing paths fail clearly without removing unrelated entries.
- `.pocketdaw` association: Windows Open With and double-click can launch Pocket DAW from a saved project.
- Second-instance open: double-clicking a different `.pocketdaw` while Pocket DAW is running focuses the existing window and opens that project.
- `pocket-daw://` handoff: Chordsmith handoff/import still succeeds after file-association testing and does not re-import consumed payloads on reload.
- Audio import: native audio import and browser fallback can import a known supported WAV, place/play it when loaded, persist media metadata, and report unloaded/missing media honestly after reopen.
- MIDI import: `.mid`/`.midi` import creates media plus editable MIDI clips, plays through the preview path, saves/reopens metadata, and rejects unsupported/corrupt MIDI without crashing.
- Updater status: About/Diagnostics and Help -> Check for Updates show the installed/candidate version and either a clear no-update result or the staged newer signed release.
- Release truth: after smoke, refresh or verify `release-status.json` and `docs/CURRENT_RELEASE_STATUS.md`; older version sections in this checklist remain historical notes only.

## Native Media Reliability First Gate

Run these rows against an exact installed artifact before claiming native media reliability beyond automated/unit coverage:

- Native audio decode/import: import known WAV, FLAC, MP3, Ogg Vorbis and AIFF/AIF fixtures where available; confirm media pool metadata records source format, decoded WAV status, sample rate, channels, duration, decoder and missing/unloaded state honestly.
- Save/reopen durability: save the project, close/reopen, reload imported audio, and confirm playback/export works or reports an actionable missing/relink state.
- Decoded-cache recovery: import a non-WAV file through the native decoder, save/reopen, move or temporarily rename the original source file, then confirm Media Pool shows the missing source honestly while Reload/playback/export can recover from the project-relative decoded WAV cache.
- Collect/relink/reload: collect a native import into `project-media/`, relink to a moved source, relink a browser runtime-only item to a durable native source, reload after reopen, and confirm metadata updates without pretending browser runtime-only media is durable, carrying stale waveform/native-decoder/decoded-cache/transient data from the old source, or storing backslash/`./` project-media refs.
- Native cache: build/rebuild native cache after media/source changes and confirm diagnostics report cache state and errors clearly.
- Evidence: record installed artifact filename, SHA-256, version, commit, tester/date, audio fixture names, and pass/fail notes.

Native Media Reliability First installed smoke: Manual / Not run for this candidate. Do not claim installed native media reliability beyond automated/unit coverage until exact-artifact installed smoke is recorded in `release-status.json` and `docs/CURRENT_RELEASE_STATUS.md`.
